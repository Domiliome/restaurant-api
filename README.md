# 🍽️ Restaurant API

Современное и быстрое REST API для управления меню ресторана, построенное на **FastAPI**, упакованное в **Docker** и использующее **SQLite** для хранения данных.

Проект разработан с учетом лучших практик: модульная структура, валидация данных, автоматическая документация и изолированное тестирование.

---

## 🚀 Технологический стек

* **Язык:** Python 3.11
* **Фреймворк:** FastAPI (ASGI)
* **База данных:** SQLite
* **Контейнеризация:** Docker & Docker Compose
* **Тестирование:** Pytest + TestClient
* **Валидация:** Pydantic v2

---

## 📂 Структура проекта

```text
restaurant-api/
├── app/                        # Логика приложения
│   ├── main.py                 # Точка входа и lifespan приложения
│   ├── database.py             # Инициализация и подключение к SQLite
│   ├── models/                 # Схемы валидации Pydantic
│   └── routers/                # Эндпоинты API (маршрутизация)

├── docker-compose.yml          # Управление контейнерами (Hot Reload включен)
├── requirements.txt            # Зависимости проекта
└── README.md                   # Документация проекта
```

## 🛠️ Запуск проекта

Для запуска проекта вам понадобится установленный **Docker Desktop**.

### 1. Клонирование репозитория
```bash
git clone https://github.com
cd restaurant-api
```

### 2. Запуск в Docker
Разверните и запустите приложение одной командой (контейнер запустится в фоновом режиме):
```bash
docker compose up --build -d
```

Приложение станет доступно по адресу: `http://localhost:8000`

---

## 📖 Документация API

После запуска Docker интерактивная документация Swagger UI доступна по адресу:
👉 **[http://localhost:8000/docs](http://localhost:8000/docs)**

### Основные эндпоинты:
* `GET /menu/` — Получить весь список блюд
* `GET /menu/{dish_id}` — Получить информацию о конкретном блюде по ID
* `POST /menu/` — Добавить новое блюдо в меню (с валидацией цены и категорий)
* `DELETE /menu/{dish_id}` — Удалить позицию из меню

---

## 🧪 Тестирование

Тесты запускаются в изолированной временной базе данных прямо внутри контейнера Docker, не нарушая целостность основных данных приложения.

Для запуска тестов выполните:
```bash
docker compose exec web pytest
```

---
### ⚙️ Разработка локально (без Docker)

Если вы хотите запустить проект локально в виртуальном окружении:
```bash
python -m venv .venv
source .venv/Scripts/activate  # Для Windows Git Bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```
